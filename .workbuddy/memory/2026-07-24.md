## 经验（可复用，关键）：modal dismiss 自定义视差转场 = 黑屏/卡死元凶
- **铁律**：tweak 的手势返回里，**modal(`presentingViewController`) dismiss 绝不能自己把底层 presenting 页面的 view 塞进转场容器(UIViewContextTransitioning.containerView)做位移/视差**。presenting 页面由 UIKit 的 `UIPresentationController` 管理，强行介入必破坏层级 → 黑屏或无法操作。
- 正确做法：modal dismiss 走系统原生动画（无跟手视差），或（方案B，未实施）只移动**被 dismiss 的 fromView(sheet 本身)**、**绝不碰 toView(presenting)**。
- nav pop 的视差可保留（toView 是 nav 堆栈标准 VC，UIKit 管理干净）。
- 诊断技巧：多版本混测时，靠「是否出现某次提交才加的日志行」区分新旧版现场；`presenting=1` 即 modal 路径、`nav.childCount>1` 即 nav 路径，是定位转场问题的第一把刀。

## 主功能"体验不好"诊断 + 提交阈值修复（commit a3be85a，日志 oback_debug(4).log）
- **现象**：手势能触发、动画也跑，但返回**几乎每次都取消**，用户反复滑仍回不去。
- **诊断（铁证在日志）**：`endTransition:` 提交判定 `(_currentPercent > commitRatio) || (vel > commitVelocity)`，
  `commitRatio=0.40 / commitVelocity=500`。微信现场 7 次 pop 仅 1 次成功：成功那次是 5 秒慢拖到 0.61；
  其余 6 次取消的进度都在 **0.14~0.38**（自然内滑只拖到约 1/3 屏），全低于 0.40 → 取消；速度也都没过 500。
  即旧阈值太严，正常"部分拖动"被判取消 = 主功能手感差的根因。
- **修复（a3be85a）**：`commitRatio 0.40→0.30`（让约 1/3 屏的拖动即可提交）、`commitVelocity 500→400`（一般甩动也能提交）。
  0.14 这类极小拖动仍取消（用户基本没动），符合预期。参数仅在 `ObackParams.defaults` 设，无硬编码覆盖（ObackPreferences 只从已存偏好读覆盖）。
- 推送：`4545e7c..a3be85a main -> main`，CI 版本 `0.1.0+a3be85a`。**待真机验证**：重装后微信/普通 App 内滑返回大多能成功提交（不再 6/7 取消）。
- **次要可能（本次未改，若仍不一致再调）**：`ObackPanGestureRecognizer.touchesMoved` 前 8pt 若纵向>横向即 `StateFailed`，
  略带纵向的起滑可能"起不来"；如需更跟手可放宽该方向判定阈值。

## 弹窗 dismiss 视差恢复（方案B，commit 73531bd，已 push：244b2af..73531bd）
- **背景**：用户"视觉上没有之前好"→ 明确指「弹窗返回动画」。8243bb8 为修黑屏把弹窗 dismiss 换成系统原生(无视差)，是近期唯一手势相关视觉降级。
- **方案B 实现（73531bd，已 push，CI 版本 0.1.0+73531bd）**：
  - `ObackTransition.h/m` 给 `ObackAnimator`/`ObackInteractiveTransition` 加 `BOOL parallaxToView`（init 默认 YES=nav pop 视差）。
    `OBApplyParallax` 增参：parallaxToView=NO 时**只移动被 dismiss 的 fromView(sheet 滑出+轻微缩小 8%)**，
    绝不碰底层 presenting(toView，黑屏根因)、也不加深遮罩(避免已可见背景闪暗)。
  - `ObackManager.beginTransition` 弹窗分支：重建手势驱动交互转场——创建 `ObackInteractiveTransition`(parallaxToView=NO)、
    显示边缘胶囊、注册 `ObackTransitioningDelegate`(`original` 保留 App 原有 modal 动画)、`top.transitioningDelegate=td` 后 dismiss。
    设置 `currentParallaxToView`(供 Tweak.xm delegate 创建 ObackAnimator 时读取置 NO)。
  - **MRC 安全关键点**：UIKit `transitioningDelegate` 是 assign 不 retain → 让被 dismiss 的 VC 经
    `objc_setAssociatedObject(top, kObackTDKey, td, RETAIN)` 自己 retain td，避免 cancel 后弹窗仍 present、
    manager 释放 td 致野指针；同一仍 present 弹窗再次内滑时复用 td(不重复释放)。endTransition 不再 release currentTD。
- **待办**：push 已成功（244b2af..73531bd），CI 应出 `0.1.0+73531bd`。**待真机验证**：弹窗从边缘内滑 dismiss 应带 sheet 滑出视差(无黑屏)。
